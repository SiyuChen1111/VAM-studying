You are helping clean and reorganize the file structure of this repository **without breaking active analysis or training workflows**.

This repository contains age-group LIM / Flanker analysis work using:
- shared Stage 1 visual evidence

- matched young control branch (`20-29 matched`)
- old group branch (`80-89`)

There are now many result directories created at different stages of the project, and the user wants the structure to be easier for both humans and later agents to understand.

## Your job

Your task is to help **organize, reduce confusion, and document** the current file structure.

You are **not** being asked to redesign the science or rerun training. Your task is mainly information architecture and safe cleanup.

## Critical constraints

### 1. Do NOT break active script paths
Many scripts still write to their original result directories. Therefore:
- do **not** move or rename original result directories in a destructive way
- prefer creating a new **organized layer** or copying / linking key outputs into clearer folders

### 2. Do NOT delete anything unless it is clearly redundant and documented
If you believe something should be deleted, first classify it as one of:
- active/current
- frozen summary
- legacy interim reference
- duplicated artifact
- obsolete/generated by mistake

If deletion is proposed, document:
- exact path
- why it is safe to delete
- what canonical replacement file exists

### 3. Preserve scientific traceability
The user must be able to tell:
- which files are based on human-only data
- which files are current model-side summaries
- which files are old interim references
- which files are not suitable for final interpretation

### 4. Do NOT mix evidence levels
Be explicit that the following are different:
- human-side final behavioral evidence
- current-best response-supervision summaries
- older trajectory preview figures
